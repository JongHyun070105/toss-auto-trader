# mydocs

이 디렉터리는 Batter's Eye 저장소에 맞춘 **Hyper-Waterfall Lite** 작업 기억 저장소다.

목표:
- 긴 채팅 대신 저장소 안에 작업 의도/계획/검증/보고를 남긴다.
- 큰 작업은 `orders/ → plans/ → reports/` 흐름으로 추적한다.
- 다음 세션이나 다른 에이전트도 같은 markdown 산출물을 읽고 이어받을 수 있게 한다.

## 구조
- `orders/` — 오늘 진행할 일과 상태 보드
- `plans/` — 작업 계획서 / 구현 계획서
- `reports/` — stage 보고서 / 최종 보고서
- `manual/` — 저장소 맞춤 운영 규칙
- `_templates/` — 문서 템플릿

## 최소 운영 규칙
1. 작은 UI 카피 수정처럼 자명한 작업은 즉시 처리 가능.
2. 구조 변경, 새 흐름, 외부 연동, 아키텍처 영향 작업은 아래 순서를 따른다.
   - `orders/YYYYMMDD.md`에 작업 등록
   - `plans/task-*.md` 작성
   - 승인 게이트 이후 구현
   - `reports/`에 stage 또는 final 보고 남김
3. 승인 게이트는 **방향 전환 / 범위 확대 / 대규모 코드 수정 전**에 반드시 거친다.
4. AI는 실행을 맡고, 사람은 우선순위/방향/품질 결정을 맡는다.

## Node 패키지 정책
- `npm install` 사용 금지
- 재현 가능한 설치가 필요하면 항상 `npm ci` 사용
- shell 스크립트와 문서 예시도 동일 규칙을 따른다
