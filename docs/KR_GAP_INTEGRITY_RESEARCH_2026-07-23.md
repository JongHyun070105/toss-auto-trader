# 국내 갭 전략 기준가 무결성 감사

작성일: 2026-07-23
대상 전략: `robust_gap5_stop0225_take12`

## 결론

새로운 수익 조건은 채택하지 않았다. 시장 가드, 종목 가격, 거래량, 후보 순위,
손절 `-2.25%`, 익절 `+12%`, 15:20 잔여 청산은 그대로 유지한다.

대신 다음 데이터 무결성 안전장치를 실전 스캔에 추가했다.

```text
전일 종가 대비 당일 공식 시가 raw 갭이 -31% 미만이면 후보에서 제외
실전 진입 가능 raw 갭 범위: -31% 이상, -5% 이하
```

이 조건은 예상수익을 높이기 위한 알파 필터가 아니다. KRX의 일반 가격제한폭은
당일 기준가 대비 `±30%`지만, 증자·배당·분할·병합·감자·재상장 같은 상황에서는
전일 종가 대신 조정 기준가로 당일 가격 범위가 정해진다. 정리매매처럼 일반
가격제한폭이 적용되지 않는 특수 거래도 있다. 따라서 raw
`당일 시가 / 캐시된 전일 종가 - 1`이 `-31%` 미만이면 일반적인 갭하락 반등과
같은 사건으로 볼 수 없다.

현재 Toss/Naver 경고 필터도 정리매매 등을 차단한다. 새 하한은 경고 데이터가
늦거나 누락되거나, 로컬 전일 종가와 Toss 당일 기준가의 비교 기준이 달라지는
경우를 막는 방어선이다.

## 고정 기준

`-31%`는 백테스트 수익을 최대화해 고른 숫자가 아니다.

1. KRX 일반 가격제한폭 `-30%`를 규칙 경계로 사용했다.
2. 호가 단위 반올림으로 정상 하한가가 잘못 제외되지 않도록 1%p 여유를 뒀다.
3. `-31%` 미만은 매수 기회가 아니라 기준가 비비교 또는 특수 거래 플래그로
   분류했다.

## 검증 범위

- 기간: 2011-01-01~2026-07-23
- 기본 DB: 현재 유니버스 중심 198,956개 이벤트
- 상장폐지 60일 전 제외 보강 DB: 220,981개 이벤트
- 상장폐지 전체 보강 DB: 221,492개 이벤트
- 비용: base 0.35%, realistic 0.75%, harsh 1.35%, extreme 2.45% 왕복
- 고정 자금: 거래마다 10,000원, 정수 주식 수량
- 검증 구간: 2011~2018, 2019~2023, 2024~NXT 전, NXT 이후, 2026, 전체
- 강건성: 상위 수익 거래 25% 제거

세 DB에서 기존 종목 조건까지 통과한 `-31%` 미만 이벤트는 각각 6건, 7건,
10건이었다. harsh 비용에서 실제 1순위 선택이 바뀐 날은 각각 2일, 1일,
4일뿐이었다.

기본 DB 전체 손익은 `1,236원` 줄었다. 2015-09-07의 `-51.16%` 후보가
백테스트상 큰 익절이었기 때문이다. 이 불리한 결과를 포함해도 사전 정의한
네 구간은 모두 양수였고 MDD는 변하지 않았다. 따라서 하한을 수익 개선으로
설명하지 않는다.

상장폐지 전체 보강 DB의 NXT 이후에는 2025-12-18 `054630`의 raw 갭
`-79.75%` 후보가 제외되면서 당일 손익이 `+475원` 개선됐다. 이 사례도
특수 기준가 후보를 제거한 결과일 뿐 반복 가능한 알파 증거는 아니다.

재현한 결과는 아니다.

## 실전 변경

- 스캐너가 공식 당일 시가를 확인한 뒤 raw 갭 `< -31%` 후보를 제외한다.
- 제외 수를 성능 로그의 `gap_integrity_exclusions`에 기록한다.
- Discord 매수 보고에 `기준가 비비교 갭 제외 N개`를 표시한다.
- 기존 로그는 새 필드가 없어도 `0개`로 해석해 호환성을 유지한다.
- 주문 시간, 예산, 시장 가드, 경고 필터, 청산 조건은 변경하지 않는다.

## 한계

1. 일봉 OHLC는 09:01 호가 큐, 스프레드, VI 순서와 실제 체결을 재현하지 못한다.
2. 과거 날짜별 공식 기준가 시계열을 별도로 확보하지 않아 raw 갭만으로 정확한
   기업행위 종류를 식별할 수 없다.
3. 2024년 이후 데이터는 이전 연구에서도 반복 사용해 독립 홀드아웃이 아니다.
4. 상장폐지 보강 DB도 모든 과거 경고·정리매매 상태를 완전하게 복원하지 못한다.
5. 안전장치 적용 후 paper/live 로그에서 제외 건수와 차순위 후보를 계속 관찰해야
   한다.

## 재현

이 스크립트는 SQLite를 읽기 전용으로 열고 live trader나 주문 API를 import하지
않는다. 결과 JSON/Markdown은 Git에서 제외된 `data/kr_gap_integrity_audit/`에
저장한다.

```bash
PYTHONPATH=src:scripts .venv/bin/python scripts/kr_gap_integrity_audit.py \
  --start 2011-01-01 \
  --end 2026-07-23 \
  --db-path data/edge_research_universe_15y.sqlite3 \
  --db-path data/kr_foreign_microstructure_research/edge_research_with_delisted_pre60.sqlite3 \
  --db-path data/kr_foreign_microstructure_research/edge_research_with_delisted_all.sqlite3
```
