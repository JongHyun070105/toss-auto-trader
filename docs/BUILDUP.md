# Toss Auto Trader Lab 빌드업 로드맵

작성일: 2026-06-23 KST

## 1. 한 줄 정의

토스증권 Open API로 계좌/시세/주문을 연결하되, 처음에는 자동매매 수익보다 **주식·API·리스크 관리 학습**을 목표로 하는 소액 자동매매 실험실.

## 2. 왜 지금 만들 수 있는가

토스증권 Open API 페이지 기준:

- 국내/해외 주식을 하나의 API로 다룰 수 있다고 안내됨.
- REST와 WebSocket 기반이라고 안내됨.
- 기능 범위는 시세, 주문, 계좌 조회, 종목/시장정보로 안내됨.
- 예시 주문 엔드포인트는 `POST /v1/orders` 형태로 공개됨.
- 현재 페이지는 사전 신청 및 문서 링크 중심이라, 실제 개발 전 공식 문서/API 키 발급 가능 여부 확인이 필요함.

## 3. MVP 목표

초기 MVP는 **돈을 버는 봇**이 아니라 아래를 증명한다.

1. 계좌 연결 없이도 과거/실시간 가격 데이터로 전략을 시뮬레이션할 수 있다.
2. 계좌 연결 후에도 처음에는 조회 전용으로 잔고·주문 가능 금액·보유 종목을 읽을 수 있다.
3. 주문 전에는 항상 손실 한도, 주문 금액, 종목 화이트리스트, 수동 승인 게이트를 통과한다.
4. 실거래 단계에서도 10,000원 이하 테스트 자본으로 제한한다.

## 4. 추천 단계

### Phase 0 — 학습/정책 정리

- 주식 기본: 시장가/지정가, 호가, 체결, 수수료, 세금, 변동성, 슬리피지 이해
- 자동매매 기본: 전략, 백테스트, 페이퍼 트레이딩, 실거래, 리스크 관리 구분
- 금지: 처음부터 AI가 마음대로 매수/매도하는 구조

### Phase 1 — 프로젝트 골격

- Python 또는 TypeScript 중 선택
- `.env` 기반 시크릿 관리
- `config.yaml`에 최대 주문 금액, 일일 손실 한도, 허용 종목 목록 작성
- SQLite로 가격/주문/시그널 로그 저장

### Phase 2 — 페이퍼 트레이딩

- 실제 주문 없이 가상 잔고 10,000원으로 매수/매도 기록
- 전략 1개만 구현: 예) 정해진 종목을 일정 간격으로 소액 분할매수하거나, 단순 이동평균 조건 실험
- 결과 지표: 승률보다 손실폭, 최대 낙폭, 거래 횟수, 수수료 반영 후 손익

### Phase 3 — Toss API 조회 연결

- 인증/토큰 발급 확인
- 계좌 목록/잔고/주문 가능 금액 조회
- 시세 REST 조회
- WebSocket 실시간 시세는 이후 단계

### Phase 4 — 주문 드라이런

- 실제 주문 API 호출 직전까지 payload 생성
- 주문 전 체크리스트 출력
- `DRY_RUN=true` 기본값 유지

### Phase 5 — 소액 실거래

- 사용자가 명시적으로 승인할 때만 `LIVE_TRADING=true`
- 1회 주문 금액 1,000~3,000원 수준부터 시작
- 일일 최대 주문 횟수 제한
- 당일 손실 한도 도달 시 자동 중지

## 5. 먼저 의논할 결정

1. 언어: Python이 빠른 실험에 유리, TypeScript는 웹 대시보드/Node 생태계에 유리.
2. 대상: 국내주식만 먼저 할지, 해외주식도 포함할지.
3. 전략: 분할매수형 학습봇 vs 기술지표 실험봇 vs 알림/분석봇.
4. UI: CLI만 할지, 나중에 작은 웹 대시보드를 붙일지.
5. 실거래 시점: 페이퍼 트레이딩 며칠/몇 회 검증 후 허용할지.

## 6. 추천 기본 선택

- 언어: Python
- DB: SQLite
- 시작 모드: CLI + 페이퍼 트레이딩
- 전략: 단일 종목/ETF 감시 + 조건 만족 시 가상 매수
- 실거래: API 조회 연결과 페이퍼 검증 후 별도 승인
